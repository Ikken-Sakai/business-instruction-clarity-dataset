# コマ4: エラー分析と改善（1.5時間）

## 🎯 このコマの目標

1. 誤分類サンプルを詳細に分析する
2. 誤分類の原因を特定する
3. 改善策を立案する
4. ハイパーパラメータ調整案を作成する

## 📋 タスクリスト

### タスク1: 誤分類の詳細分析（30分）
- [ ] 誤分類サンプルをカテゴリ分けする
- [ ] パターンを見つける
- [ ] データの問題点を洗い出す

### タスク2: 可視化・レポート作成（25分）
- [ ] 誤分類の傾向をグラフ化
- [ ] 混同行列の詳細分析
- [ ] 文字数と誤分類の関係分析
- [ ] 詳細レポート作成

### タスク3: 改善策の立案（20分）
- [ ] ハイパーパラメータ調整案
- [ ] データ拡張案
- [ ] モデル構造変更案（オプション）

### タスク4: 改善案の優先順位付け（10分）
- [ ] 効果が高そうな施策をリストアップ
- [ ] 実装の難易度を評価
- [ ] 次のコマで実施する施策を決定

### タスク5: 調整後の設定ファイル作成（5分）
- [ ] 新しいCONFIGを作成
- [ ] 実行計画書を作成

## ✅ 完了条件

- [ ] 誤分類パターンが明確になった
- [ ] `error_analysis_report.md` が作成された
- [ ] 改善策が3つ以上立案された
- [ ] 次の学習で使うCONFIGが決まった

## 📂 成果物

- `03_error_analysis.py` - エラー分析スクリプト
- `error_analysis_report.md` - 詳細レポート
- `figures/error_patterns.png` - 誤分類パターンの可視化
